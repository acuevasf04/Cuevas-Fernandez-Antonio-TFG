# ¿Qué es GLPI y para qué sirve?

GLPI (Gestionnaire Libre de Parc Informático) es una plataforma de código abierto (*open source*) basada en web diseñada para centralizar toda la gestión tecnológica y el soporte técnico de una organización en un solo lugar.

---

## ¿Qué es GLPI?
Es una herramienta integral que combina dos disciplinas fundamentales de la informática (TI):
1. **ITAM (Asset Management):** Gestión y control del inventario informático.
2. **ITSM (Service Management):** Gestión de servicios de soporte y atención al usuario (Mesa de ayuda / Helpdesk).

Al ser una aplicación web desarrollada en PHP, permite a los administradores, técnicos y usuarios finales acceder a ella desde cualquier navegador, eliminando la necesidad de instalar programas adicionales para interactuar con el sistema de soporte.

---

## ¿Para qué sirve?

GLPI sirve principalmente para resolver cuatro grandes necesidades dentro de cualquier departamento de soporte o TI:

### 1. Controlar el Inventario Tecnológico (Saber qué tienes)
Permite registrar de manera detallada y automática todo el ecosistema de hardware y software de la empresa.
* **Hardware:** Saber exactamente cuántos ordenadores, servidores, pantallas, impresoras y smartphones existen, quién los tiene asignados y en qué departamento están.
* **Software y Licencias:** Controlar los programas instalados, gestionar el número de licencias disponibles y recibir alertas antes de que caduquen los contratos de soporte.
* **Redes:** Mapear las conexiones físicas, switches, routers y direcciones IP asignadas.

### 2. Gestionar el Soporte Técnico / Helpdesk (Resolver problemas)
Sirve como el canal oficial para recibir, priorizar y solucionar los reportes de los usuarios.
* **Gestión de Tickets:** Los usuarios pueden registrar incidencias (ej. *"No me funciona internet"*) o peticiones (ej. *"Necesito un ratón nuevo"*).
* **Asignación Eficiente:** El sistema dirige automáticamente el ticket al técnico idóneo según el tipo de problema.
* **Historial por Equipo:** Permite vincular los tickets directamente al ordenador o equipo afectado, ayudando a los técnicos a ver si ese dispositivo ha fallado repetidamente en el pasado.

### 3. Crear una Base de Conocimiento (Evitar trabajo repetitivo)
Sirve para almacenar soluciones a problemas comunes (sección de FAQs). Los usuarios pueden resolver dudas sencillas por sí mismos antes de abrir un ticket, y los técnicos nuevos disponen de una guía centralizada con los procedimientos de la empresa.

### 4. Automatizar y Medir el Rendimiento (Tomar decisiones)
* Genera estadísticas de cuántos problemas se resuelven a tiempo y cuáles son las fallas más comunes.
* Controla los **SLA** (Acuerdos de Nivel de Servicio) para asegurar que las incidencias críticas se atiendan con la rapidez necesaria.
